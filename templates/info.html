March 2021 This fantastic journey started on a rainy night of March 2021.
Wandering on Twitter, I stumbled upon a post of Crypto Dictionary that talked about the GOST cipher. This symmetric block cipher was the Soviet counterpart of the old American DES.
The GOST cipher is a 64 bit block symmetric cipher that is based, like DES, on a Feistel network. The key is 256 bit long, unlike DES where the key was dramatically shorter. Both encryption and decryption are performed through 32 rounds. Each round consists of bit rotations, XORs and use of S-Boxes. In this implementation, the S-BOXES are fixed according to the standard GOST R 34.12-2015.
The most effective attack against GOST requires like 270 plaintexts, so the cipher is theoretically broken. Nevertheless, I think that using it as a toy cannot be harmful for anyone. Just don't use it for your credit card data and you'll be fine.
This particular implementation uses Password-Based Key Derivation Function 2 (PBKDF2) with SHA-256 to derive a key starting from you password. PBKDF2 provides a stronger resistance to brute-force attacks.
Since I like to implement basic paper-hand ciphers using various programming languages (Python, basically), I decided that the challenge of implementing a more serious cipher would have been intriguing.
Working on this cipher for a couple of nights, I managed to make it work.
Nevertheless, the program could only be launched using the command line, so I wasn't that happy.
A couple of days later I learned about the existence of Flask and so I decided that the time had come to give a User Interface to this program. Flask is a very versatile tool and web programming CAN be fun, in the end.
This is the final result, obtained by combining Python, HTML, Javascript and some CSS (mostly Bootstrap).

March 2023 Around two years later, in March 2023, I finally decided to take a big leap of faith: leave behind the old, clumsy Python implementation of GOST and work on a brand new one in C, one of the fastest and most famous programming languages. C is much faster than Python so I was sure that a new, proper implementation would have been way faster. Things were quite hard in the beginning as I had to go through the pointers' nightmare, but I eventually managed to make it work. Now, instead of using the GOST.py module, a C shared library is used to perform encryption and decryption.
So, after two years, I can say that the final result is obtained by combining together Python, HTML, Javascript, CSS and, most importantly, C.

What do the bold words mean on the homepage?

  • Plaintext is the message that you want to encrypt to unintelligible to those who do not know the password. An example? A super cringe love letter;
  • Password is the secret password that will be used to derive a key that will encrypt your message. The strength of encryption heavily relies on this parameter, so always choose a password that is not that easy to guess;
  • Salt is a random value that helps the key derivation function to derive a unique, fresh key even if the password is the same. Salt is used in order to let a user re-use the same password again and again without relying on the same key;
  • Operating mode (or mode of operation) is a specific algorithm that makes a block cipher behave like a stream cipher, enabling a user to encrypt messages that are longer than a block. Besides ECB, which is the most simple and weak at the same time, all the other ones are good to go;
  • IV (initialization vector) is a block-sized, fresh and unique quantity that is used by modes of operations to guarantee certain properties to the ciphertext. IVs should never be used more than once, at least on short periods. Completely random ones are always the best choice;
  • Ciphertext is the result of the encryption.
